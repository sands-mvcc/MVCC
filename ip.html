IP Header Illustration

The Internet Protocol (IP) is a set of rules governing the format of data sent over the Internet or other networks. IP addresses, which are used to uniquely identify devices on a network, are part of this protocol. IP is responsible for addressing and routing packets of data so that they can travel across networks and arrive at the correct destination.

Version (4 bits)

Indicates the version of the IP protocol. For IPv4, this value is 4. Back to header

IHL (4 bits)

Internet Header Length, the length of the IP header, measured in 32-bit words. Back to header

DSCP (6 bits)

DiffServ Code Point, used for packet classification. Back to header

ECN (2 bits)

Explicit Congestion Notification, used for indicating network congestion. Back to header

Total Length (16 bits)

The total length of the IP packet (header + data), in bytes. Back to header

Identification (16 bits)

Identifies the fragments of a single IP packet. Back to header

Flags (3 bits)

Control flags for fragmenting packets. Back to header

Fragment Offset (13 bits)

Indicates the position of the fragment in the original packet. Back to header

Time to Live (8 bits)

Indicates the maximum time the packet is allowed to remain in the network. Back to header

Protocol (8 bits)

Indicates the next level protocol used in the data portion of the IP datagram. Back to header

Header Checksum (16 bits)

A checksum on the header to ensure integrity. Back to header

Source Address (32 bits)

Specifies the sender's IP address. Back to header

Destination Address (32 bits)

Specifies the receiver's IP address. Back to header

Options + Padding (Variable length)

Optional fields used for various purposes, and padding to ensure the header is a multiple of 32 bits. Back to header
