STP Header Illustration

The Spanning Tree Protocol (STP) is a network protocol that builds a loop-free logical topology for Ethernet networks. The basic function of STP is to prevent bridge loops and the broadcast radiation that results from them. STP allows a network design to include backup links to provide fault tolerance if an active link fails, without the risk of bridge loops, or the need for manual enabling/disabling of these backup links.

Protocol ID (16 bits)

Identifies the protocol as STP. The value is typically 0x0000. Back to header

Version (8 bits)

Indicates the version of the STP protocol. Back to header

BPDU Type (8 bits)

Indicates the type of BPDU. Common values are 0x00 for Configuration BPDU and 0x02 for Topology Change Notification BPDU. Back to header

Flags (8 bits)

Contains various flags used for STP operations. Back to header

Root ID (64 bits)

Identifies the root bridge by its bridge priority and MAC address. Back to header

Root Path Cost (32 bits)

Indicates the cost of the path to the root bridge from the transmitting bridge. Back to header

Bridge ID (64 bits)

Identifies the bridge by its bridge priority and MAC address. Back to header

Port ID (16 bits)

Identifies the port from which the BPDU was sent. Back to header

Message Age (16 bits)

Indicates the age of the BPDU in the network. Back to header

Max Age (16 bits)

Specifies the maximum age of the BPDU before it is discarded. Back to header

Hello Time (16 bits)

Specifies the time interval between BPDU messages sent by the root bridge. Back to header

Forward Delay (16 bits)

Specifies the time a bridge should wait before changing states. Back to header
