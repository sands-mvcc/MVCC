RARP Header Illustration

The Reverse Address Resolution Protocol (RARP) is a network protocol used by a device on a local area network (LAN) to request its IPv4 address from a gateway server's Address Resolution Protocol (ARP) table or cache. This protocol is used primarily by diskless workstations, which do not have the non-volatile storage to permanently store an IP address.

Hardware Type (16 bits)

Specifies the type of hardware (e.g., Ethernet).

Protocol Type (16 bits)

Specifies the type of protocol (e.g., IPv4).

Hardware Address Length (8 bits)

Length of the hardware address (e.g., 6 for Ethernet).

Protocol Address Length (8 bits)

Length of the protocol address (e.g., 4 for IPv4).

Operation (16 bits)

Specifies the operation (e.g., request or reply).
