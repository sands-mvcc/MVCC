SMTP Header Illustration

SMTP (Simple Mail Transfer Protocol) is an internet standard for email transmission. First defined by RFC 821 in 1982, it was last updated in 2008 with the Extended SMTP additions by RFC 5321, which is the protocol in widespread use today. SMTP provides a set of commands that authenticates and directs the transfer of email from one mail server to another.

HELO (Maximum 512 characters)

Identifies the client to the server. Back to header

MAIL FROM (Maximum 512 characters)

Specifies the sender's email address. Back to header

RCPT TO (Maximum 512 characters)

Specifies the recipient's email address. Back to header

DATA (Variable length)

Indicates that the following lines constitute the message data. Back to header

RSET (Maximum 512 characters)

Resets the current mail transaction. Back to header

VRFY (Maximum 512 characters)

Verifies the existence of a mailbox. Back to header

EXPN (Maximum 512 characters)

Expands a mailing list. Back to header

NOOP (Maximum 512 characters)

Requests a no-operation response from the server. Back to header

QUIT (Maximum 512 characters)

Terminates the SMTP session. Back to header

220 (Maximum 512 characters)

Service ready. Back to header
