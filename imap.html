IMAP Header Illustration

IMAP (Internet Message Access Protocol) is an internet standard protocol used by email clients to retrieve messages from a mail server. IMAP allows users to view and manage their email directly on the mail server without downloading it to their local device. This protocol supports multiple clients connected to the same mailbox, allowing for better email synchronization across different devices.

IMAP Commands

LOGIN (Maximum 1024 characters)
SELECT (Maximum 1024 characters)
FETCH (Maximum 1024 characters)
SEARCH (Maximum 1024 characters)
STORE (Maximum 1024 characters)
COPY (Maximum 1024 characters)
NOOP (Maximum 1024 characters)
LOGOUT (Maximum 1024 characters)

IMAP Responses

OK (Maximum 512 characters)
NO (Maximum 512 characters)
BAD (Maximum 512 characters)
PREAUTH (Maximum 512 characters)
BYE (Maximum 512 characters)

LOGIN (Maximum 1024 characters)

Authenticates the user by specifying the username and password. Back to header

SELECT (Maximum 1024 characters)

Selects a mailbox to access its messages. Back to header

FETCH (Maximum 1024 characters)

Retrieves specific messages or message data from the selected mailbox. Back to header

SEARCH (Maximum 1024 characters)

Searches for messages in the selected mailbox that match the specified criteria. Back to header

STORE (Maximum 1024 characters)

Modifies message data in the selected mailbox. Back to header

COPY (Maximum 1024 characters)

Copies specific messages to another mailbox. Back to header

NOOP (Maximum 1024 characters)

Requests a no-operation response from the server to keep the connection alive. Back to header

LOGOUT (Maximum 1024 characters)

Terminates the IMAP session. Back to header

OK (Maximum 512 characters)

Indicates that the command was successful. Back to header

NO (Maximum 512 characters)

Indicates that the command was not successful. Back to header

BAD (Maximum 512 characters)

Indicates a protocol error or command syntax error. Back to header

PREAUTH (Maximum 512 characters)

Indicates that the connection was pre-authenticated. Back to header

BYE (Maximum 512 characters)

Indicates that the server is terminating the connection. Back to header
