DHCP Header Illustration

The Dynamic Host Configuration Protocol (DHCP) is a network management protocol used on Internet Protocol (IP) networks for automatically assigning IP addresses and other communication parameters to devices connected to the network. DHCP automates and centrally manages these configurations, allowing devices to join the network with no manual configuration.

Operation Code (8 bits)

Indicates the message type: 1 for request, 2 for reply. Back to header

Hardware Type (8 bits)

Specifies the type of hardware used for the local network. For Ethernet, this is 1. Back to header

Hardware Length (8 bits)

Specifies the length of the hardware address. For Ethernet, this is 6. Back to header

Hops (8 bits)

Set to 0 by a client before transmitting a request and used by relay agents to control the forwarding of messages. Back to header

Transaction ID (32 bits)

A randomly chosen number used by the client and server to associate messages and responses between a client and a server. Back to header

Seconds (16 bits)

Elapsed time in seconds since the client began the address acquisition or renewal process. Back to header

Flags (16 bits)

Used to control the interpretation of the "broadcast" bit in the DHCP message. Back to header

The IP address of the next server to use in the bootstrap process. Back to header

Relay Agent IP Address (32 bits)

The IP address of the relay agent, if any, used by the client. Back to header

Client Hardware Address (48 bits)

The hardware address of the client. Back to header

Client Hardware Address (Padding) (64 bits)

Padding for the client hardware address to ensure alignment. Back to header

Server Name (128 bytes)

An optional server host name, null-terminated string. Back to header

Boot Filename (128 bytes)

An optional null-terminated string, used by the client to request a particular boot file. Back to header
