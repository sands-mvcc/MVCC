NTP Header Illustration

NTP (Network Time Protocol) is a networking protocol for clock synchronization between computer systems over packet-switched, variable-latency data networks. NTP was designed by David L. Mills of the University of Delaware and is one of the oldest internet protocols still in use. It synchronizes the clocks of computer systems to within a few milliseconds of Coordinated Universal Time (UTC).

Leap Indicator (2 bits)

Warns of an impending leap second to be inserted or deleted in the last minute of the current day. Back to header

Version Number (3 bits)

Indicates the NTP version number. Back to header

Mode (3 bits)

Indicates the mode of the NTP message, such as client, server, broadcast, etc. Back to header

Stratum (8 bits)

Indicates the stratum level of the local clock, with 0 being the highest level (primary reference) and increasing numbers representing lower levels. Back to header

Poll Interval (8 bits)

Indicates the maximum interval between successive NTP messages, in log2 seconds. Back to header

Precision (8 bits)

Indicates the precision of the local clock, in log2 seconds. Back to header

Root Delay (32 bits)

Indicates the total round-trip delay to the primary reference source, in NTP short format. Back to header

Root Dispersion (32 bits)

Indicates the maximum error relative to the primary reference source, in NTP short format. Back to header

Reference ID (32 bits)

Indicates the reference identifier, which depends on the stratum and type of the clock. Back to header

Reference Timestamp (64 bits)

Indicates the time when the system clock was last set or corrected, in NTP timestamp format. Back to header

Originate Timestamp (64 bits)

Indicates the time at which the request departed the client for the server, in NTP timestamp format. Back to header

Receive Timestamp (64 bits)

Indicates the time at which the request arrived at the server, in NTP timestamp format. Back to header

Transmit Timestamp (64 bits)

Indicates the time at which the reply departed the server for the client, in NTP timestamp format. Back to header
